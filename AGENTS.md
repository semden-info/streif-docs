## Про проєкт

**Streif** — Android-застосунок, що гейміфікує щоденне дослідження місцевості в Норвегії. Користувач «розкриває» власну мапу: розкриті об'єкти (будинки, стежки, вершини) засвічуються кольором на повністю видимій карті, побудованій на відкритих державних картографічних даних (Kartverket).

- **Робоча назва:** Streif (фінальний бренд — TBD; домен `streif.no` у переговорах; фолбеки: Streifa, Omvei).
- **Платформа:** Android-first (Kotlin + Jetpack Compose). iOS відкладено.
- **Позиціонування:** «Walkable для Норвегії, але глибший» — та сама механіка розкривання, але на якісніших держданих і одразу місто + природа.
- **Власник:** Денис Семенюк — solo-розробник (ENK), Ørsta/Volda, Sunnmøre.

## Ключові рішення (зафіксовані)

**Механіка.** Зафарбовування розкритих об'єктів на **повністю видимій** карті. НЕ туман (fog-of-war) — туман приховував би рельєф/небезпеки і суперечив би безпеці. Об'єкти сірі за замовчуванням → набувають кольору за типом при розкритті (UI-мова «розкрито/побачено», D6). Ціль (post-MVP): опційний «серпанок» — легка десатурація недослідженого (+ «свіжість» розкритого, P19); без приховування рельєфу.

**Скоуп.** Гібрид місто + природа — це **візія**. Виконання **поетапне**: MVP запускає один шар end-to-end, решта каркасом. Три шари: (1) Місто — будинки/bydeler; (2) Стежки — turstier, км/висота; (3) Вершини — топи/DNT-хижки. Архітектура (багатошарова модель даних) підтримує всі три з дня 1; MVP — device-local через CLI-пайплайн tiles без backend/БД (PostGIS — лише v2, `DECISIONS.md` D12).

**MVP-виконання (зафіксовано 2026-06-20, деталі в `05`).** Перший реліз — **місто-first** (провізорно; формальний лок у `09`) і **device-local** (без логіну/живого backend — швидше тестувати). Рендер зафарбовування — **двошаровий** (сіра база + visited GeoJSON-overlay; Android-`feature-state` недоступний → двошаровий єдиний, D7/D22). Дані будинків — відкриті **INSPIRE Core2d (геометрія) + Matrikkelen (тип), CC-BY 4.0**, напряму з Geonorge (`building2osm` WFS зламаний — D8; spike тимчасово на OSM).

**Після Фази-0 (review + тех-стек-пас, 2026-06-21).** Backend **прибрано з MVP** (CLI-пайплайн tiles, без БД); матчинг уточнено (буфер сегмента, accuracy, eligibility «доступний з пішої мережі»); метрики — три виміри **Coverage/Variety/Discovery** (без score); spike **розділено** (симульований GPS → реальний FGS); UI-мова «**розкрито**»; реframe — «**мапа новизни**», метрика успіху = «**новий маршрут**». **Канонічний журнал рішень — `docs/DECISIONS.md`.**

**Spike-2 (2026-06-22…26, деталі в `CHANGELOG.md`).** Дві зміни архітектури за польовими даними:
- **D24 — on-demand area download** (замінює фіксований бандл): `BuildingStore` інкрементальний, `AreaLoader` тягне будинки в bbox навколо GPS на льоту + локальний кеш. **Тест-джерело = Runtime Overpass (OSM/ODbL)**; ⚠️ **перед публічним релізом ОБОВ'ЯЗКОВО мігрувати на pre-hosted CC-BY CDN** (надійність + масштаб + ліцензія). NB: це **новий вихідний мережевий запит** (bbox-координати до Overpass) — сирий GPS-трек усе одно не залишає пристрій (D14), але потік зон задокументовано в `07`.
- **D25 — матчинг до КОНТУРУ будинку (edge-distance) + closest-approach gate** (замінює **матчинг-частину** D5 — центроїд-радіус; ActivityGate-частина D5 чинна): центроїд спричиняв і «зарано», і пропуск через дорогу/видовжених. `R_FAR=18`/`MIN_EDGE=8`/`SLACK=1.5`, bbox-grid, локальна проєкція. Польово-верифіковано.

**Планування за конкурентним аналізом (2026-06-27, деталі — `08`/`09`/`DECISIONS.md`).** З огляду 20 аналогів: ніша «розкривання повністю видимої мапи за типом об'єкта» у Норвегії **вільна**; найближчий конкурент за гейміфікацією — DNT/UT (SjekkUT); фандинг-прецедент — Sparebanken Møre→StikkUT (2 млн kr). Нові ухвалені: **D26** deferred auth (логін після цінності), **D27** POI «цікаві місця» (читати з відкритих даних → особисті локальні нотатки → публічні відгуки v2) — у природному шарі, **D28** мінімалістична візуальна мова (контурні іконки колір-за-типом, без перевантаження, collect/unlock **без абстрактного score**), **D29** UX точності GPS (не розкривати тихо), **D30** захист від ретроактиву. Топ-болі жанру (наш «не наступити») → `09` §11.

**Гейміфікація — кор + опційний шар (D33, 2026-07-11).** Замість «спокій vs гейміфікація» — **спокійний кор** (дефолт, MVP; D13/D20) **+ опційний шар, що вмикається** (post-MVP): діорама (2D-first), типи-як-точки-дій, баланс місто↔природа **без формули/score**, новизна маршруту, історичний шар, **свіжість** (розкрите тьмяніє/освіжається при revisit). Гарди: безпека не opt-in, без втрати прогресу (D30), без streak/FOMO. Механіки → `04` §11 / `DECISIONS.md` **D33, P19 + P21–P25**; Coverage-% = **per tettsted (P20)**. Брейнсторм `04-gamification2.md` влито в `04`.

**Розкривання об'єктів — 2 рівноправні стовпи Місто/Природа, механіка за геометрією (D34, 2026-07-12; дизайн-сесія → codex-звуження → Nature-v1 points-only ЗБУДОВАНО, інкр. 1-4 + фото).** Кор ambient-reveal + «збирай»-механіка (turmål/POI/вершини/хижки з відкритих даних; стежки=Turrutebasen) — розширення ідеї «наповнення прогулянок цікавими місцями» (Stikk UT-стиль). Геометрія→механіка: площа=edge(D25) · лінія=стежки «пройшов уздовж» (10%+«пройдено»@75%, **ще не збудовано**) · точка=proximity(+фото/нотатка). **Іконка режиму** (тап=ручне перемикання). **Збудовано v1:** POI-точки (рендер+картка+фото device-verified; **proximity-зарахування польово підтверджено** — 3 зарахування 2026-07-13 за 4 хв на ходу). Обидва режими на **тест ASAP** (safety=release-gate, не test-gate). Дані verified (Turrutebasen+OSM), НЕ скрейпити бази Stikk UT/MoroTur. 8 ухвал + черга + курація-адмінка(v2) → `DECISIONS.md` **D34** / `03` §C-D / `CHANGELOG` 2026-07-12.

**Аудиторія.** Первинна (dogfood) — сам Денис (місто + біг + гори в одній людині). Вторинні — бігун, новоприбулий/іммігрант, сім'я з дітьми. Зростання — локальна спільнота Sunnmøre / Møre og Romsdal.

**Принципи:**
- *Ядро (day-1):* offline-first (архітектурна вимога), локалізація (bokmål + nynorsk + англійська), privacy-by-design (MVP device-local — сирий GPS не залишає пристрій, D14; на сервер — лише v2, агрегований стан «розкрито», не сирі треки), battery management.
- *Gate (тільки природний шар):* безпека — varsom.no + yr.no + disclaimer + кнопка екстреного виклику. Природний шар НЕ виходить у продакшн без цього. Не блокує міський MVP.
- *Дух:* спокій за замовчуванням, челенджі опційні, без streak-шантажу. Доросла аудиторія.
- *Accessibility:* uu-tilsynet / WCAG 2.1 AA — поступово, юридична вимога в Норвегії.

**Соціальне.** М'які друзі + локальні лідерборди, opt-in, **pseudonymous** (нікнейм за замовчуванням, реальне ім'я ніколи не обов'язкове). Без real-time location sharing. **Реалізація — post-MVP (v2)** (причина — обсяг роботи для solo-dev, не GDPR).

**Монетизація.** НЕ комерційний продаж. Норвезька модель суспільно-корисного проєкту: **sponsorship** (sparebank-stiftelser напр. Sparebanken Møre, fylkeskommune, friluftsråd) + **donations**. Рішення свідомо відкладене, без тиску на MVP.

**Успіх.** Локальна спільнота в сотні активних користувачів у Sunnmøre / Møre og Romsdal. Особистий dogfooding.

## Технічний стек (планований — деталі в `05`)

**Android:** Kotlin + Jetpack Compose · MapLibre GL Native · Room (SQLite) · WorkManager + Foreground Service · Hilt · Retrofit + Kotlinx Serialization · Health Connect API.

**Backend (лише v2 — НЕ в MVP):** Laravel 11 + Filament 3 · PostgreSQL + PostGIS · Sanctum · Redis · FCM. *MVP — без backend; tiles генеруються CLI-пайплайном (`DECISIONS.md` D12).*

**Дані:** Kartverket — будинки з відкритих **INSPIRE Buildings Core2d + Matrikkelen-Bygningspunkt** (CC-BY 4.0; геометрія Core2d + тип Matrikkelen, експорт напряму з Geonorge — `building2osm` WFS зламаний, `DECISIONS.md` D8); висоти з høydedata.no. *NB: класичний FKB-Bygning — Norge digitalt-обмежений (платний доступ), не використовуємо.* + OpenStreetMap Norway (ODbL) — тест-джерело on-demand. **Доставка будинків — on-demand завантаження зон навколо GPS (D24)**, не фіксований бандл: тест = Runtime Overpass (ODbL), ⚠️ продакшн = pre-hosted CC-BY (INSPIRE+Matrikkelen) на CDN. *(Tippecanoe→PMTiles лишається для seed перф-режимів.)* Базова карта: Kartverket WMTS (`topograatone`). Стежки/вершини/хижки: Kartverket (turrutebasen, Stadnamn, topodata) + DNT. **POI (Nature-v1, D34):** OSM (куратний allowlist) + **Wikidata P18 → Wikimedia Commons** для ілюстративних фото місць (CC-BY-SA; ⚠️ Commons вимагає описовий User-Agent інакше 403; per-image кредит автор+ліцензія — pre-release TODO). *(Деталі → `05`/`06`.)*
