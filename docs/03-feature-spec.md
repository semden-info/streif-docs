# Streif — Feature Spec (Специфікація функцій)

> **Статус:** Чернетка v0.3 — оновлено 2026-07-11 (звірено з `DECISIONS.md`; включає Spike-2 D24/D25 і post-MVP D33/P19–P23)
> **Спирається на:** `01-product-vision.md`, `02-user-personas.md`
> **Призначення:** повний каталог того, що продукт *може* робити. Точний скоуп першого релізу (яка підмножина) → `09-mvp-scope.md`. Деталі гейміфікації → `04-gamification.md`.

---

## Легенда пріоритетів

- **[CORE]** — фундамент механіки; потрібен будь-якому релізу, незалежно від обраного шару.
- **[MVP]** — кандидат у перший реліз (фінальне рішення в `09`, залежить від того, який шар запускаємо першим).
- **[GATE]** — обов'язкова умова релізу свого шару (не виходить без цього).
- **[v2]** — після MVP.
- **[LATER]** — пізніше / якщо буде сенс.
- **[POST-MVP]** — після MVP-1, поза v2-обсягом (напр. опційний шар гейміфікації, `04` §11).
- **[ongoing]** — наскрізна вимога, супроводжується постійно (не разова фіча).

> **Важлива залежність:** який шар запускається першим — ✅ вирішено: **місто-first** (`09`, `DECISIONS.md` D3). Фічі шарів нижче марковані за логікою «що потрібно, *якщо* цей шар у релізі».

## A. Ядро механіки (cross-layer)

- **[CORE] Карта.** MapLibre GL Native, базовий шар Kartverket (завжди повністю видимий — рельєф, стежки, назви). Центрування на користувачі.
- **[CORE] Зафарбовування об'єктів.** Дискретні об'єкти приглушено-сірі за замовчуванням → набувають кольору за типом **при розкритті** (двошарово: сіра PMTiles-база + visited-overlay через `GeoJsonSource`/`setGeoJson`; деталі рендеру й відомі обмеження → `05` §3.1, `DECISIONS.md` D21). Стан зберігається.
- **[CORE] GPS-трекінг.** Foreground service з activity recognition (walking/still), щоб не тримати високоточний GPS постійно (батарея).
- **[CORE] Реєстрація «розкрито».** Матчинг GPS до **контуру** eligible-об'єкта (за відстанню до стіни, не до центроїда). Розкриття спрацьовує на **closest-approach gate** — коли користувач проминув найближчу точку або підійшов упритул. Фільтри: accuracy + vehicle-gate (гістерезис). Результат → «**розкрито**» в локальній БД. (D25 замінив центроїд-радіус D5; eligibility D6; дані будинків — on-demand, D24.)
- **[CORE] Offline-first.** Локальна БД (Room) як джерело істини; мапа й building-tiles доступні офлайн. **MVP — device-local без backend** (D12); опортуністичний sync на сервер — `[v2]`.
- **[CORE] Базовий профіль і статистика.** % дослідження, лічильники розкритих об'єктів, проста зведена сторінка.
- **[LATER] Інтенсивність візитів.** Насиченість кольору росте з повторними візитами (heatmap-логіка) — глибина для щоденного використання, але розмиває ідею «нове», тож не на старті.
- **[LATER] «Серпанок».** Опційна легка десатурація недослідженої зони (підсилення драйву відкриття) — поверх зафарбовування, конфігурований перемикач. (Рельєф ніколи не ховається — див. Vision §4.)

## B. Шар «Місто»

- **[MVP] Зафарбовування будинків і bydeler.** Полігони будинків (**OSM-геометрія + Matrikkelen тип** — комбінований ODbL-тайл; продакшн-джерело → `06`/`DECISIONS.md` D31) засвічуються; райони мають свій % дослідження.
- **[MVP] Типи будівель.** Кольорове кодування за matrikkel-типом (житло / hytte / церква / …) — чого не роблять ані Walkable, ані Fog of World / CityStrides.
- **[MVP] % дослідження району / міста.** Розбиття на bydeler з окремими відсотками.
- **[LATER] Рідкісні типи / колекції будівель.** Окремий лік для незвичних типів (маяки, stavkyrkje тощо).

## C. Шар «Стежки»

- **[MVP] Зафарбовування turstier.** Marked stier (Kartverket) засвічуються при проходженні.
- **[MVP] Метрики руху.** Кілометри й метри висоти (те, чого нема в Stikk UT).
- **[v2] Strava / Garmin sync.** Біг/похід рахується без окремого запуску Streif (персони Eirik + Денис). Інтеграція, не ядро.

## D. Шар «Вершини / природа»

- **[MVP] Колекція вершин.** База топів (Kartverket Stadnamn + topodata); вершина «засвічується» при досягненні. Колекції (Sunnmørsalpane, Volda-fjell, >1000 moh тощо).
- **[MVP] DNT-хижки.** Хижки як окрема колекція відвідувань.
- **[LATER] Vertical meters lifetime.** Сумарний вертикальний набір за рік + еквіваленти (Everest 8848 m).
- **[GATE] Безпека.** Цей шар **не виходить у продакшн без**: інтеграції varsom.no (лавини/паводки) + yr.no (погода) + стартового disclaimer + кнопки екстреного виклику (113 / Hovedredningssentralen). Hard gate (Vision §7).

## E. Гейміфікація (високорівнево — деталі в `04`)

- **[MVP] Досягнення / бейджі.** Базовий набір (перше відвідування, віхи покриття).
- **[MVP] Колекції.** Вершини, типи будівель, райони — як набори для «закриття».
- **[v2] Сезонні цілі.** Опційні самоцільні планки (як Stikk UT-сезон, але необов'язкові).
- **[v2] Щоденні міні-місії.** AI-генерована маленька ціль («пройди ці 3 квартали, які ще не бачив»).
- **[MVP] Метрики прогресу.** Три виміри — **Coverage / Variety / Discovery** (без єдиного score/XP, без формул балів). Деталі → `04`. *(Замінило старе «формули балів / difficulty» — `DECISIONS.md` S6/D13.)*
- **[POST-MVP] Опційний шар гейміфікації** (D33) — спокійний кор за замовчуванням + опційний шар, що вмикається: діорама (2D-first), типи-як-точки-дій, баланс місто↔природа **без формули**. Деталі → `04` §11 / `DECISIONS.md` P19–P23.
- **[POST-MVP] Новизна маршруту** («нова вулиця сьогодні») — інструментує ★-метрику успіху (новий маршрут, D2); залежить від шару пройдених вулиць (P6). → `04` §11.2 / `DECISIONS.md` P22.
- **[POST-MVP] Історичний шар реальних місць** — розкриття історії відвіданих місць. → `04` §11.2 / `DECISIONS.md` P23.

## F. Соціальне

- **[v2] Pseudonymous акаунти.** Нікнейм за замовчуванням, реальне ім'я ніколи не обов'язкове. Логін: email-only або Google/Apple Sign-In, мінімум даних.
- **[v2] Друзі (opt-in).** Додавання друзів, перегляд їхнього прогресу — лише за згодою обох.
- **[v2] Локальні лідерборди (opt-in).** Показ у лідерборді завжди добровільний; без агресивних нотифікацій.
- **[LATER] Hidden Gems.** Користувацькі улюблені точки (видові місця, gapahuk, badeplasser) — соціальний шар поверх карти.
- *Принцип:* без real-time location sharing (ніколи). Деталі приватності → `07-privacy-gdpr.md`.

## G. Family-режим

- **[v2] Реєстрація за дітей.** Батьки реєструють прогулянку за молодших (як familiemedlem у Stikk UT). Персони: сім'я Aas + Денис із донькою.
- **[v2] Дитячий UI.** Спрощений режим — великі іконки, «казкові місця» й наклейки замість «% покриття».

## H. Cross-cutting / нефункціональне

- **[CORE] Локалізація.** bokmål + nynorsk + англійська з MVP.
- **[CORE] Privacy by design.** MVP device-local (сирий GPS не залишає пристрій, D14); на сервер (лише `[v2]`) — агрегований стан «розкрито», не сирі треки. → `07`.
- **[CORE] Battery management.** Activity recognition + fused location; не тримати високоточний GPS постійно.
- **[MVP] Onboarding.** Перший екран з «ah-ha» моментом — напр. одразу показати твою місцевість («Ось твоя Ørsta. N будинків. Скільки ти бачив?»).
- **[ongoing] Accessibility.** uu-tilsynet / WCAG 2.1 AA — закладати поступово від першого екрана (контраст, тач-таргети, screen-reader мітки). Юридична вимога в Норвегії.

## Відкриті точки / залежності

- **Який шар перший у MVP** — ✅ **вирішено: місто-first** (`09`, `DECISIONS.md` D3).
- **Acceptance criteria** для [CORE] і [MVP] фіч — детальні критерії пропишемо у `09` (або окремих spec), коли скоуп зафіксовано. Тут — каталог, не контракт.
- **Карта vs власні tiles** — базовий шар Kartverket WMTS чи власні vector tiles (pmtiles) — технічне рішення → `05` / `06`.
- **Default-мова інтерфейсу** (за пристроєм vs норвезька) → фаза дизайну.
