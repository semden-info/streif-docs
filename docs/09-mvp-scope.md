# Streif — MVP Scope (Скоуп MVP)

> **Статус:** Чернетка v0.2 — оновлено 2026-07-11 (Фаза-0 + Spike-2: D24 on-demand, D25→D25.1 матчинг, анти-патерни §11, opt-in шар гейміфікації D33)
> **Спирається на:** `01`–`08` + `DECISIONS.md`
> **Призначення:** **формально зафіксувати скоуп MVP** — який шар, що в/поза, acceptance criteria. Наріжний доказовий документ перед розробкою. MVP розбито на два кроки: **MVP-0 (гола механіка)** і **MVP-1 (спокійна гейміфікація + дизайн)**.

---

## 1. Рішення: який шар перший

**Зафіксовано: місто-first.** Природний шар — друга фаза.

Обґрунтування (підсумок `01`–`06`):
- **Перевірена механіка** — зафарбовування будинків (ядро в стилі Walkable), найменший продуктовий ризик.
- **Щоденне використання** — місто Volda/Ørsta — первинний патерн Дениса + персони Oksana/сім'я.
- **Дані доступні зараз** — INSPIRE Core2d + Matrikkelen (CC-BY 4.0) для Volda/Ørsta (`06`).
- **Без safety-GATE** — місто не тягне varsom/yr/SOS, тож шлях до продакшну коротший (`01` §7).
- CLAUDE.md і так націлений сюди (імпорт OSM-будинків Volda/Ørsta, building-tiles як перший spike).

## 2. Визначення MVP + два кроки

**«MVP готовий»** = робочий end-to-end зріз по місту, придатний до dogfood Денисом і родиною на реальному пристрої.

| Крок | Суть | Мета |
|---|---|---|
| **MVP-0** | Гола механіка: карта + сірі будинки + зафарбовування під час ходьби + грубий прогрес | Найшвидший шлях до тесту на пристрої; всередині — технічний spike |
| **MVP-1** | Додає спокійну гейміфікацію (`04`) + фінальний дизайн (`08`) | Приємний dogfood, повна «спокійна» петля |

Принцип розбиття (за лінією «fast-first»): **MVP-0 = петля працює і її можна тестувати**; усе, що робить її *приємною/гейміфікованою/відполірованою*, — це **MVP-1**.

## 3. MVP-0 — гола механіка (in-scope)

- **Карта** — Kartverket WMTS `topograatone` (сіра база).
- **Будинки** — доставка **on-demand навколо GPS** (D24; продакшн-дефолт = R2-CDN тайли OSM+Matrikkelen/D31, фолбек = Overpass). *(PMTiles-бандл лишається лише як seed для перф-режимів спайку.)*
- **Зафарбовування** — двошаровий overlay (`05` §3.1); колір розкритих **рудиментарний** (єдиний або грубо за типом) — фінальна палітра за типами → MVP-1 після `08`.
- **GPS-трекінг** — FGS `type=location`, foreground-only, Fused Location (`05` §4).
- **Guard проти авто** — Activity Recognition (підказка) + швидкісний фільтр **із гістерезисом** (`05` §4, `DECISIONS.md` D5).
- **Реєстрація «розкрито»** — **edge-матчинг до контуру + closest-approach** (`05` §5, `DECISIONS.md` D25, замінив центроїд-радіус D5) → стан «**розкрито**» в Room. Рахуються лише **eligible** будинки (доступні з публічної пішої мережі — дешевий фільтр, D6, на MVP-0).
- **Базовий прогрес** — лічильник розкритих будинків + грубий % покриття.
- **Device-local** — без логіну/runtime-backend; механіка працює offline **для вже завантажених/кешованих зон** (нові зони довантажуються при наявності мережі, D24).
- **Мінімальна локалізація** — UI малотекстовий; bm+nn+en як каркас.
- **UX точності GPS** (`DECISIONS.md` D29) — показувати стан надійності GPS, не розкривати/не пропускати тихо.
- **Інваріант «розкрите лишається розкритим»** (`DECISIONS.md` D30) — апдейти даних не знімають уже розкрите (на Room-персистенції).

> **Критерій MVP-0:** Денис ходить Ørsta → будинки фарбуються в реальному часі → видно прогрес. Технічний spike (двошаровий рендер на пристрої) валідовано всередині цього кроку.

## 4. MVP-1 — спокійна гейміфікація + дизайн (in-scope)

- **Фінальні кольори за типами будівель** (після `08`).
- **Візуальна мова** (`DECISIONS.md` D28) — прості контурні іконки колір-за-типом, низька щільність маркерів, collect/unlock без абстрактного score (`08` §14).
- **Метрики прогресу** — три виміри **Coverage / Variety / Discovery** (`04` §3, `DECISIONS.md` D13).
- **Колекції** — типи будівель + райони (`04` §4); **% per tettsted** (P20).
- **Бейджі** — перші кроки + віхи покриття (`04` §5).
- **Святкування віх** — 25/50/75/100% без тиску (`04` §6).
- **М'який тижневий підсумок** (`04` §7).
- **Onboarding ah-ha** — «Ось твоя Ørsta. N будинків. Скільки ти бачив?» (`03` §H); **deferred auth** (`DECISIONS.md` D26) — пропонувати збереження прогресу після розкриття частини.
- **Повна локалізація** bm+nn+en.

> **Критерій MVP-1:** dogfood-приємний; спокійна гейміфікація з `04` працює; **жодного score/streak**.

## 5. Поза MVP (обидва кроки) — post-MVP

## 6. ★ Acceptance criteria

| Фіча | Крок | Критерій «зроблено» |
|---|---|---|
| Двошаровий рендер (spike) | MVP-0 | Кольорування рендериться/оновлюється **плавно (без видимого джанку, ~30+ fps)** на реальному mid-range Android при кількасот–низьких тисячах visited; `setGeoJson` поза main-thread, латентність прийнятна. **#1 ризик — зняти ДО UI.** |
| Реєстрація «розкрито» | MVP-0 | Прохід повз **eligible** будинок по публічній вулиці (walking) → будинок «розкрито» й зафарбований; стоячи/проїздом/далеко від пішої мережі — ні. |
| Трекінг/батарея | MVP-0 | FGS стартує без `SecurityException` (Android 14+); **трекінг триває при згаслому екрані під час активного запису**; guard проти авто працює; розумна витрата батареї на типовій прогулянці. |
| Offline | MVP-0 | Трекінг + реєстрація + зафарбовування працюють без мережі; база-tiles кешуються опортуністично. |
| Прогрес | MVP-0 | Лічильник + % покриття оновлюються коректно й персистентно (Room). |
| Кольори за типами | MVP-1 | Будинки фарбуються за палітрою `08`; «невідомий/інше» має свій бакет. |
| Колекції/бейджі/віхи | MVP-1 | Набори показують прогрес; віхи святкуються без тиску (`04`). |
| Тижневий підсумок | MVP-1 | In-app картка «N нового цього тижня»; без push-тиску/streak. |
| Локалізація | MVP-1 | Повний UI у bm + nn + en. |

**Пороги матчингу (edge-eligibility + centroid-CA тайминг, D25→D25.1; польово-валідовані 2026-06-27, тюнінг триває — P2):**
- **`R_FAR`=18 м** — eligibility: стіна ≤18 → будинок «у грі» (важіль 18↔20 на ширші вулиці); **`ACC_MAX`=30 м** (fail-closed);
- **тайминг — closest-approach до ЦЕНТРОЇДА** (`SLACK_CEN`=2 м): розкрити коли проминув **середину** будинку («по середині, коли йдеш уздовж»); **`MIN_EDGE_NEAR`=3 м** — впритул-байпас; `moved`-guard проти GPS-джитера на місці;
- **vehicle-фільтр** — Activity Recognition як підказка + швидкісний гістерезис (`DECISIONS.md` D5), польово-валідовано (98% блок).
- *Метод валідації:* прогулянки Дениса + ✓/✗-маркування (debug) → `spike/fieldtest/analyze.py` звіряє з реально пройденим → тюнінг порогів. Replay D25.1: розкриття медіана 3.3 м від центру (79% у межах 5 м).

## 9. Метрики успіху MVP

- **★ Головна метрика (після review):** чи Streif **реально спонукає обрати новий маршрут** — частка нових сегментів проти повторних. Це і є доказ цінності (`01` §9, `DECISIONS.md` D2).
- **Dogfood** — Денис (і донька) користуються регулярно; механіка приємна в щоденному циклі.
- **Технічно** — петля «йдеш → фарбується» стабільна на реальному пристрої.
- **Розширення** — перші beta-користувачі Sunnmøre (`01` §9). Без числових KPI-планок — спокійний дух.

## 10. Ризики й відкриті точки

**Ризики:**
- **Двошаровий рендер** — головний; знімається spike-кроком (§8, крок 1).
- **Ліцензія footprint-полігонів INSPIRE** — `06` §11; блокує **публічний реліз**, не dogfood.
- **Kommunenummer** — валідувати перед хардкодом.

**Відкриті точки:**
- **Тюнінг порогу `R_FAR` (18↔20)** — edge-метод D25 польово-валідовано; лишається підкрутити на ширші вулиці за новими ✓/✗-мітками (`DECISIONS.md` P2).
- *(Джерело меж для Coverage-% — ✅ вирішено **P20**: SSB **Tettsteder** (Volda/Ørsta не мають офіційних bydeler); дрібніше grunnkrets — пізніше, з системою досягнень.)*
- **Default-мова інтерфейсу** (за пристроєм vs норвезька) → фаза дизайну / `08`.
- **Міграція on-demand Overpass → CC-BY CDN** — обов'язково перед публічним релізом (`DECISIONS.md` D24/P4, `06` §11).

## 11. Анти-патерни жанру (чого уникати — з аналізу 20 конкурентів, 2026-06-26)

Найгучніші 1-зіркові теми жанру = наш список «не наступити». Більшість уже покрита рішеннями — нижче мапа «біль → наш guard»:

| Анти-патерн (чий біль) | Наш guard |
|---|---|
| **Зупинка трекінгу у фоні / OEM-killers** (Fog of World впав до 3.6 на Play; Turf, Run An Empire) | Тест на Samsung/Xiaomi/Huawei (Фаза 6), acceptance «трекінг при згаслому екрані» (§6). #1 пріоритет. |
| **Витрата батареї** (Geocaching, Munzee, Pokémon GO) | Явний перф-таргет (Stage C); офлайн-first без серверного чату (D15). Планку задає Walkable — спершу перевірити, чи в нього continuous vs significant-change GPS. |
| **Paywall на ядро / subscription-creep** (Pokémon GO, Walkable, Geocaching, Strava) | Ядро безкоштовне назавжди; модель sponsorship+donations (CLAUDE.md). |
| **Втрата прогресу** (Walkable — втрата року; Run An Empire) | Інваріант «розкрите не втрачається/не мис-кредититься» + локальний бекап (D11, D30). |
| **Залежність від live-backend / third-party API** (Wandrer↔Strava; Fog of World↔Google Drive CASA $9k) | Device-local MVP (D12/D14). ⚠️ on-demand Overpass→CC-BY CDN не зробити новою крихкістю (D24/P4). |
| **Деградація рендера на mid-range після MapLibre** (StreetComplete v59 — той самий стек) | Gate ДО UI на реальному mid-range (D22, §6, Фаза 6). |
| **Ретроактивна зміна completion** від апдейтів даних (CityStrides/OSM re-break) | Стабільний `building_id` + інваріант (D30). |
| **Тиск/змагання/сезони/idle-економіка** (Turf раунди, Run An Empire coin-walls, Pokémon daily-login) | Свідомо не робимо — спокій, без score/streak (D13/D20). |
| **Cold-start / порожня мапа** (Turf hand-placed zones, Pokémon rural, Walkable передмістя) | On-demand дані (D24); перевірити щільність даних поза центром. |
| **Тихий провал точності GPS** (Munzee, Turf, Wandrer) | Surface accuracy-state (D29). |
| **GPS-spoofing / cheating** (Turf, Pokémon GO) | Релевантно лише за v2-лідербордів — врахувати при scoping соц (`10` §11). |
| **Поламані базові жести мапи** (Strava drag/zoom, Turf auto-follow) | Table-stakes — drag/zoom/auto-follow не ламати (`08`). |
