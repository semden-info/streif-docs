> edge-матчинг (D25) реалізовані. Поточний фокус ретесту — **перевірити фікс матчингу D25**
> (чи зникло «зарано» + чи ловляться будинки через дорогу) і **зібрати ✓/✗-мітки** для тюнінгу.
> Передумова: Pixel 9 під'єднаний по USB. Claude драйвить install/логи через adb.

## 0. Підготовка (Claude робить, коли телефон під'єднаний)
- `install -r app-debug.apk`
- `pm clear no.streif.spike` — **чистий старт** (стирає старі логи + кеш зон → справжній on-demand-тест).
- видати дозволи: `pm grant … ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION / POST_NOTIFICATIONS / ACTIVITY_RECOGNITION`.
- `battery.ps1 start` (Stage C) — якщо міряємо батарею тією ж прогулянкою (кабель ще під'єднаний).
- `svc power stayon true` — лише для стаціонарної перевірки; для реальної прогулянки/батареї **навпаки** від'єднати.
- ⚠️ **Увімкни мобільні дані** — on-demand тягне зону з мережі (кеш порожній після `pm clear`).

## 1. Прогулянка-тест (чи механіка працює наживо)
1. Відкрий **Streif**. Сіра мапа твоєї місцевості (де б ти не був — зона довантажиться) + «Розкрито: N…» + **«Старт прогулянки»**.
2. Тапни **Старт**. Перший раз — дозвіл геолокації: **«При використанні застосунку»** + **точне** місцезнаходження.
3. З'явиться сповіщення «Streif — прогулянка» (FGS) + синя точка «ти тут».
4. **Пройдись** звичним темпом. Будинки мають **засвічуватись кольором, коли ти з ними порівнявся** (а не за ~10 м до того), включно з тими **через дорогу**.
5. **Стоп** (у застосунку або сповіщенні) → **закрий і знову відкрий** → розкриті лишаються кольоровими (збереження).

## 2. Чек-ліст спостережень (головне для D25)
- 🎯 **«Зарано»:** будинки досі засвічуються ДО того, як ти до них дійшов? (мало стати ~при порівнянні, не за 10 м.)
- 🎯 **«Через дорогу»:** будинки на ІНШІЙ стороні вулиці, повз яку йдеш, тепер розкриваються?
- ⚠️ **Хибні розкриття:** засвітилось щось, повз що ти НЕ йшов (далеко збоку)?
- ⚠️ **Стоячи на місці** — нові будинки не мають сипатись (хіба ти впритул ≤8 м до стіни).
- 📏 Статус показує **точність (м)** і **швидкість (м/с)** — глянь типову точність у місті/під деревами.

## 3. ★ Маркування ✓/✗ (головний інструмент тюнінгу — debug)
> Тапни по будинку на ходу, щоб дати точний фідбек. Цикл: **тап → ✓ зелена (правильно) → тап → ✗ червона (неправильно) → тап → зняти**. Кожен тап логиться у `marks.csv`.
- **✓** — застосунок має рацію (правильно зафарбував / правильно лишив сірим).
- **✗** — застосунок помилився: або **дарма зафарбував** (зарано/чуже), або **дарма НЕ зафарбував** (пропуск через дорогу — тапни сірий будинок, який мав розкритись).
- Особливо мітьте сумнівні: кутові, за парканом, надто рано/пізно, протилежний бік широкої вулиці.

## 4. Vehicle-gate (Stage B — готовий, можна перевірити)
- Проїдь шматок на авто з увімкненим трекінгом — будинки вздовж дороги **не** мають розкриватись (gate блокує). Статус покаже «авто/велосипед — пауза».
- (Польово вже підтверджено: 98% швидких фіксів блокуються; додаткова перевірка не обов'язкова.)

## 5. Battery-gate (Stage C — фізично від'єднаний; можна тією ж прогулянкою)
> Заряджання через USB спотворює замір — треба реальний анплаг (`battery.ps1`, не `dumpsys ... unplug`).
1. Claude: `battery.ps1 start` (кабель під'єднаний).
2. **Від'єднай кабель.** Старт → **30–60 хв** ходьби (екран можеш гасити — FGS тримає GPS; маркування потребує екран-он, тож число вийде «змішане»).
3. Повернись, під'єднай кабель → `battery.ps1 stop` (→ %/год).
- **Орієнтир:** ~8–12 %/год при згаслому екрані (Pixel 9) — синтетичний поріг. Повторити 2× (старт-заряд 60–80%).
