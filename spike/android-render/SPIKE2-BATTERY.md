# Spike-2 Stage C — battery-gate (інструкція)

> Мета: чи трекінг прийнятний по батареї на реके прогулянці (`11` Spike 2 gate).
> Скрипт: `spike/battery.ps1` (start/stop). Заряд міряємо **ФІЗИЧНО ВІД'ЄДНАНИМ** —
> заряджання через USB спотворює замір (`dumpsys battery unplug`/`stayon` НЕ допомагають).

## Порядок (one-command з кожного боку)

1. **Кабель під'єднаний.** Claude/ти:
   ```
   powershell -File spike/battery.ps1 start
   ```
   → скидає `batterystats`, фіксує рівень+час.
2. **Від'єднай кабель.** Відкрий **Streif** → «Старт прогулянки».
3. **Гуляй 30-60 хв** звичним темпом. **Екран можна гасити** (FGS тримає GPS). Не запускай інші важкі застосунки (щоб не змішати розряд).
4. Повернувся → **«Стоп»** у застосунку → **під'єднай кабель**.
5. Одразу:
   ```
   powershell -File spike/battery.ps1 stop
   ```
   → друкує **%/год**, mAh-розряд (з `batterystats`), тривалість.

## Присуд
- Орієнтир: **~8-12 %/год при згаслому екрані** на Pixel 9 (синтетичний, офіційного нема).
- **Повторити 2-3×** (той самий маршрут, старт-заряд 60-80%, дати охолонути) → усереднити.
- Окремо глянути **screen-on** (верхня межа) проти screen-off.
- `diag.csv` за ту саму прогулянку — крос-чек (скільки фіксів, чи gate тримався).

## Що далі за результатом
- Якщо забагато — тюнити **GPS-інтервал** (зараз 2с → 3-5с?) + перевірити Activity Recognition-вартість.
- ⚠️ Pixel 9 — флагман; для продакшн-присуду — **дозамір на mid-range** (`11` §7).
